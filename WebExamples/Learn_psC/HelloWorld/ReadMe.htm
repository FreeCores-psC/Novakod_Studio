First program: Hello World!

This example shows how the classic “Hello World!” program is coded in psC.

Program structure

In C++ there is a main() function. In psC there is a main component and one or more sub-components. Sub-components communicate with signals. As shown on the figure, in this example, there is one sub-component, CHello, and two signals connecting the external ports iConsole  and oConsole to the component corresponding ports.


Here is how this is coded:

component main(

    in  ubyte iConsole,                // <<< External ports >>>

    out ubyte oConsole)

{

    component CHello(

        in  active ubyte iConsole,             // <<< Sub-component ports >>>

        out active ubyte oConsole)

    {

        // <<< Code of CHello component >>>

    };

 

    // Process declaration

    main::CHello PHello;     // <<< The process PHello is an instance

                             //         of the component CHello >>>

 

    // Signal declaration

    ubyte sConsoleIn  = {PHello.iConsole, iConsole};

    ubyte sConsoleOut = {PHello.oConsole, oConsole};

};

Constant string declaration

String constants are declared as follows:

    const string cHelloWorld = "Hello Novakod World!\n";

Memory declaration

In C++ there is a global data memory. In psC, since resources must be minimized, you declare only the memory you need, here is an example where an array of 22 ubytes is declared:

    ubyte memory

    {

        sHelloWorld[22]

    };  

ARDUINO like operation

The basic execution model in Arduino is based on two functions: setup() and loop().

Ø setup() function

The setup() function is executed once at startup.

    uint:5 i;

   

    setup()

    {

        // Need to copy the constant string to memory

        i = (uint:5)0;

 

        while(cHelloWorld[i] != 0 )

        {

            sHelloWorld[i] = cHelloWorld[i]; 

            i++;

        }

        sHelloWorld[i] = 0ub; // End of string

        return;

    }    

Ø loop() function

The loop() function, as its name implies, is continuously executed.

    loop()

    {

        // Wait for a char to start

        while(!istrig(iConsole)) {};

       

        // Call puts() as in C++

        call puts(sHelloWorld);

       

        // As for Arduino, loops forever

        // Type another character to repeat

    }   

StdioLib - psC version of stdio.h

Here is a simplified version of the library “StdioLib”, corresponding to the C++ library “stdio.h”. Here, we have one function “puts()” to write a string to the port “oConsole”, stdout in C++.

library StdioLib

{

    // These ports must be declared

    //       in the component calling puts()

    extern in  active ubyte iConsole;

    extern out active ubyte oConsole;

 

    function puts(ubyte& str)

        // Local variable declaration

        uint:5 i

    {

        i = 0;

        while(str[i] != 0)  

        {         

            oConsole := str[i];

            i++;

        };

        return;

    }  

};
